## Folder Convention

Use one subfolder per project.

Recommended naming style:

- lowercase
- hyphen-separated
- descriptive enough to explain the task and model

Examples:

- `placement-prediction-logistic-regression`
- `house-price-prediction-random-forest`
- `customer-churn-xgboost`

## Recommended Structure

Each project folder can contain:

- a notebook or source code
- dataset files used by that project
- a project-specific README
- saved model artifacts if needed

## Notes

- Keep small practice projects together in this workspace.
- Give a separate Git repository to any project that is portfolio-ready, deployable, or meant to be shared independently.
- If multiple projects share common utilities, consider a separate shared folder later.
